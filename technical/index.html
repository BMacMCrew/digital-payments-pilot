Core Components Explained

1. SPR Builder & Validator

  • Assembles Standard Payment Requests from PO, invoice, receipt data
  • Validates CGAC/TAS codes and budget authority
  • Performs 3-way match (PO = Invoice = Receipt)
  • Generates immutable blockchain transaction

2. Compliance Engine

  • Enforces acquisition regulations (FAR/DFARS)
  • Validates financial obligations and commitments
  • Checks anti-deficiency protections
  • Audits all transactions for improper payment indicators

3. Blockchain Ledger

  • Stores immutable transaction records from appropriation to disbursement
  • Creates cryptographic proof chain for all payment actions
  • Enables atomic settlements across multiple parties
  • Provides real-time audit trail for Treasury and agency CFOs

4. Smart Contracts

  • IF/THEN rules encoded on blockchain
  • Automated execution when conditions met
  • Example: IF (3-way match PASS AND budget available) THEN (generate SPR)
  • Reduces manual approvals by 95%+

5. Atomic Settlement Engine

  • All-or-nothing execution for multi-party payments
  • Coordinates across FedNow, RTP, ACH, card networks
  • Guarantees financial integrity (no partial failures)
  • Immutable settlement records on blockchain

Integration Guide

ERP Integration Flow

  1. Purchase Order Creation: Agency creates PO in their ERP system
  2. Data Sync: Actus receives PO data via Ascella integration layer
  3. Invoice Receipt: Supplier submits invoice (via IP or direct)
  4. 3-Way Match: Actus validates PO = Invoice = Receipt
  5. SPR Generation: Actus creates blockchain-backed SPR
  6. Treasury Submission: SPR flows through ERP to Treasury BFS/PAM
  7. Approval Callback: Treasury PAM returns approval to Actus
  8. Payment Execution: Actus triggers Fiserv for settlement
  9. Webhook Notification: Settlement confirmation updates ERP

Supported ERP Systems

Integration is provided through our partner Ascella, who specializes in federal ERP connectivity:

  • SAP: Direct API integration via Ascella connector
  • Oracle EBS: Web services integration
  • Momentum: Batch file exchange with real-time webhooks
  • Legacy Systems: SFTP/CSV import with validation layer

Integration Timeline

Typical ERP integration takes 6-8 weeks:

  • Weeks 1-2: Schema mapping and field validation with Ascella
  • Weeks 3-4: Sandbox testing with sample data
  • Weeks 5-6: UAT with agency stakeholders
  • Weeks 7-8: Production cutover and parallel run

Security & Compliance

Authentication

  • JWT Tokens: All API requests require valid JWT bearer token
  • Token Expiry: 1 hour default, refresh tokens valid 30 days
  • Scopes: Fine-grained permissions (read:payments, write:spr, etc.)

Data Encryption

  • In Transit: TLS 1.3 for all API communications
  • At Rest: AES-256 encryption for all stored data
  • Blockchain: Cryptographic signatures for transaction integrity

PCI Compliance

Payment Card Security

For G2B virtual card payments:

  • Never store raw PANs in your systems
  • Use Actus tokenization for all card references
  • Implement hosted payment fields for sensitive data capture
  • Log card usage but mask all but last 4 digits

FedRAMP Authorization

  • Level: FedRAMP High (in progress)
  • Controls: 421 NIST 800-53 controls implemented
  • Logging: Centralized audit logs with 7-year retention
  • Monitoring: 24/7 SOC with automated threat detection

Blockchain Security

  • Consensus: Byzantine Fault Tolerant (BFT) algorithm
  • Nodes: Distributed across Treasury, agencies, and Actus infrastructure
  • Immutability: Hash chaining prevents retroactive modification
  • Access Control: Smart contract permissions enforce who can execute transactions
